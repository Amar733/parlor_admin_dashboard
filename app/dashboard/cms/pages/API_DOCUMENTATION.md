# Pages API Documentation

This document describes the API endpoints that need to be implemented on your backend server for the Pages management functionality.

## Base URL
All endpoints should be prefixed with your API base URL: `{{base_url}}/api/pages`

## Endpoints

### 1. Get All Pages
**Method:** `GET`  
**URL:** `/api/pages`  
**Description:** Retrieve all pages  

**Response Example:**
```json
[
  {
    "_id": "64f1a2b3c4d5e6f7g8h9i0j1",
    "name": "About Us",
    "slug": "about-us",
    "content": "<p>This is the about us page content with <strong>rich text</strong> formatting.</p>",
    "position": "header",
    "status": "Active",
    "createdAt": "2024-01-15T10:30:00Z",
    "updatedAt": "2024-01-15T10:30:00Z"
  }
]
```

### 2. Get Page by Slug
**Method:** `GET`  
**URL:** `/api/pages/{slug}`  
**Example:** `/api/pages/about-us`  
**Description:** Retrieve a specific page by its slug  

**Response Example:**
```json
{
  "_id": "64f1a2b3c4d5e6f7g8h9i0j1",
  "name": "About Us",
  "slug": "about-us",
  "content": "<p>This is the about us page content with <strong>rich text</strong> formatting.</p>",
  "position": "header",
  "status": "Active",
  "createdAt": "2024-01-15T10:30:00Z",
  "updatedAt": "2024-01-15T10:30:00Z"
}
```

### 3. Create New Page
**Method:** `POST`  
**URL:** `/api/pages`  
**Headers:** `Content-Type: application/json`  

**Request Body:**
```json
{
  "name": "About Us",
  "slug": "about-us",
  "content": "<p>This is the about us page content with <strong>rich text</strong> formatting.</p>",
  "position": "header",
  "status": "Active"
}
```

**Response Example:**
```json
{
  "_id": "64f1a2b3c4d5e6f7g8h9i0j1",
  "name": "About Us",
  "slug": "about-us",
  "content": "<p>This is the about us page content with <strong>rich text</strong> formatting.</p>",
  "position": "header",
  "status": "Active",
  "createdAt": "2024-01-15T10:30:00Z",
  "updatedAt": "2024-01-15T10:30:00Z"
}
```

### 4. Update Page
**Method:** `PUT`  
**URL:** `/api/pages/{id}`  
**Example:** `/api/pages/64f1a2b3c4d5e6f7g8h9i0j1`  
**Headers:** `Content-Type: application/json`  

**Request Body:**
```json
{
  "name": "Updated About Us",
  "slug": "updated-about-us",
  "content": "<p>Updated content with <em>rich text</em> formatting.</p>",
  "position": "footer",
  "status": "Inactive"
}
```

**Response Example:**
```json
{
  "_id": "64f1a2b3c4d5e6f7g8h9i0j1",
  "name": "Updated About Us",
  "slug": "updated-about-us",
  "content": "<p>Updated content with <em>rich text</em> formatting.</p>",
  "position": "footer",
  "status": "Inactive",
  "createdAt": "2024-01-15T10:30:00Z",
  "updatedAt": "2024-01-15T12:45:00Z"
}
```

### 5. Delete Page
**Method:** `DELETE`  
**URL:** `/api/pages/{id}`  
**Example:** `/api/pages/64f1a2b3c4d5e6f7g8h9i0j1`  

**Response Example:**
```json
{
  "message": "Page deleted successfully"
}
```

## Data Schema

### Page Object
```typescript
interface Page {
  _id: string;           // MongoDB ObjectId
  name: string;          // Display name of the page
  slug: string;          // URL-friendly identifier (unique)
  content: string;       // Rich text HTML content
  position: string;      // "header" | "footer" | "sidebar"
  status: string;        // "Active" | "Inactive"
  createdAt: string;     // ISO date string
  updatedAt: string;     // ISO date string
}
```

## Validation Rules

### Required Fields
- `name`: Must be a non-empty string
- `slug`: Must be a non-empty string, URL-friendly (lowercase, hyphens only)
- `content`: Can be empty string but must be present
- `position`: Must be one of: "header", "footer", "sidebar"
- `status`: Must be one of: "Active", "Inactive"

### Unique Constraints
- `slug`: Must be unique across all pages

### Content Handling
- The `content` field contains HTML generated by the rich text editor
- Ensure proper HTML sanitization on the backend to prevent XSS attacks
- Support for common HTML tags: `<p>`, `<strong>`, `<em>`, `<ul>`, `<ol>`, `<li>`, `<a>`, `<img>`, etc.

## Error Responses

### 400 Bad Request
```json
{
  "error": "Validation failed",
  "details": {
    "name": "Name is required",
    "slug": "Slug must be unique"
  }
}
```

### 404 Not Found
```json
{
  "error": "Page not found"
}
```

### 500 Internal Server Error
```json
{
  "error": "Internal server error",
  "message": "Database connection failed"
}
```

## Implementation Notes

1. **Database**: Use MongoDB with a `pages` collection
2. **Authentication**: All endpoints should require valid JWT authentication
3. **Permissions**: Ensure only admin users can create/update/delete pages
4. **Slug Generation**: Auto-generate slugs from page names if not provided
5. **Content Sanitization**: Sanitize HTML content to prevent XSS attacks
6. **Indexing**: Create indexes on `slug` and `status` fields for better performance

## Frontend Integration

The frontend Pages management interface is located at:
- **URL**: `/dashboard/cms/pages`
- **Component**: `app/dashboard/cms/pages/page.tsx`
- **Rich Text Editor**: Uses ReactQuill for WYSIWYG editing
- **API Integration**: Uses the centralized API configuration from `config/api.ts`

Test all endpoints with tools like Postman or curl:

```bash

curl -X GET "{{base_url}}/api/pages" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"

# Create a new page
curl -X POST "{{base_url}}/api/pages" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -d '{
    "name": "Test Page",
    "slug": "test-page",
    "content": "<p>Test content</p>",
    "position": "header",
    "status": "Active"
  }'
```
